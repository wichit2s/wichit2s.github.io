Flutter Architecture Patterns

MVVM & Clean Architecture

  • MVVM: UI (View) → reactive binding ↔ ViewModel → uses UseCases/Repositories.
  • Clean Architecture: Presentation ↔ Domain ↔ Data with dependency rule inward.

Default State Management

Key Issues with setState

  • State tightly coupled with widgets → hard to reuse logic.
  • Rebuilding large widget trees on every change can hurt performance.
  • Testing business logic separately is difficult.
  • Difficult to manage state across multiple screens or modules.
  • No structured approach → app scales poorly as complexity grows.

Example: Counter App using setState


class CounterPage extends StatefulWidget {
  @override
  _CounterPageState createState() => _CounterPageState();
}

class _CounterPageState extends State {
  int count = 0;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('Count: $count', style: TextStyle(fontSize: 24)),
            Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                IconButton(
                  icon: Icon(Icons.remove),
                  onPressed: () => setState(() => count--),
                ),
                IconButton(
                  icon: Icon(Icons.add),
                  onPressed: () => setState(() => count++),
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}

Takeaway: Works for simple apps but becomes messy for complex state flows, multi-screen updates, or modular projects. This motivates the use of advanced state management solutions like BLoC, GetX, or Riverpod.

State Management in Flutter

Comparison: BLoC, GetX, Riverpod

BLoC

  • Event → State streams
  • Highly structured, testable
  • Steep learning curve

GetX

  • Reactive & lightweight
  • Built-in navigation & DI
  • Easy learning curve

Riverpod

  • Provider’s evolution
  • Type-safe & compile-time safety
  • Good balance of flexibility & structure

State Flow: BLoC vs GetX vs Riverpod

Example: Counter App with BLoC 


// 1. Define Events
abstract class CounterEvent {}
class Increment extends CounterEvent {}
class Decrement extends CounterEvent {}

// 2. Define State
class CounterState {
  final int count;
  CounterState(this.count);
}

// 3. Create BLoC
class CounterBloc extends Bloc {
  CounterBloc() : super(CounterState(0)) {
    on((event, emit) => emit(CounterState(state.count + 1)));
    on((event, emit) => emit(CounterState(state.count - 1)));
  }
}

// 4. UI Layer
BlocBuilder(
  builder: (context, state) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('Count: ${state.count}', style: TextStyle(fontSize: 24)),
        Row(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            IconButton(
              icon: Icon(Icons.remove),
              onPressed: () => context.read().add(Decrement()),
            ),
            IconButton(
              icon: Icon(Icons.add),
              onPressed: () => context.read().add(Increment()),
            ),
          ],
        ),
      ],
    );
  },
);

Flow: User taps → Event → BLoC → new State → UI rebuilds.

Example: Counter App with Riverpod 


// 1. Define a StateNotifier
class CounterNotifier extends StateNotifier {
  CounterNotifier() : super(0);

  void increment() => state++;
  void decrement() => state--;
}

// 2. Create a Provider
final counterProvider = StateNotifierProvider((ref) {
  return CounterNotifier();
});

// 3. UI Layer
class CounterPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final count = ref.watch(counterProvider);

    return Scaffold(
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text('Count: $count', style: TextStyle(fontSize: 24)),
            Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                IconButton(
                  icon: Icon(Icons.remove),
                  onPressed: () => ref.read(counterProvider.notifier).decrement(),
                ),
                IconButton(
                  icon: Icon(Icons.add),
                  onPressed: () => ref.read(counterProvider.notifier).increment(),
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}

Flow: User taps → StateNotifier updates state → ConsumerWidget rebuilds automatically.

Dependency Injection & Modularization

Key Concepts

  • Decouple class dependencies for testable, maintainable code.
  • Use get_it or Provider for service location.
  • Organize project into modules: Features → Domain → Data → Presentation.
  • Follow single responsibility and SOLID principles.

Example: get_it Setup


// 1. Register services
final getIt = GetIt.instance;

void setup() {
  getIt.registerLazySingleton(() => ApiService());
  getIt.registerFactory(() => CounterController());
}

// 2. Use anywhere
class MyWidget extends StatelessWidget {
  final counterController = getIt();

  @override
  Widget build(BuildContext context) {
    return Text('Count: ${counterController.count}');
  }
}

Flow: Services and controllers are injected on demand, reducing tight coupling and improving modularity.

Navigation 2.0 & Deep Linking

Key Concepts

  • Navigation 2.0 separates URL state from UI widgets.
  • Supports **web-style routing**, deep linking, and nested navigation.
  • Pages managed via RouterDelegate and RouteInformationParser.
  • Enables **dynamic routing** for large apps and multi-role dashboards.

Example: Simple Navigation 2.0


// 1. Define pages
class MyRoutePath { final String page; MyRoutePath.home() : page = 'home'; }

// 2. RouteInformationParser
class MyRouteParser extends RouteInformationParser {
  @override
  Future parseRouteInformation(RouteInformation ri) async {
    if (ri.location == '/about') return MyRoutePath()..page = 'about';
    return MyRoutePath.home();
  }
}

// 3. RouterDelegate
class MyRouterDelegate extends RouterDelegate
    with ChangeNotifier, PopNavigatorRouterDelegateMixin {

  String currentPage = 'home';

  @override
  Widget build(BuildContext context) {
    return Navigator(
      pages: [
        if (currentPage == 'home') MaterialPage(child: HomePage()),
        if (currentPage == 'about') MaterialPage(child: AboutPage()),
      ],
      onPopPage: (route, result) {
        currentPage = 'home';
        notifyListeners();
        return route.didPop(result);
      },
    );
  }

  @override
  Future setNewRoutePath(MyRoutePath path) async {
    currentPage = path.page;
  }
}

Flow: URL changes → RouteInformationParser parses → RouterDelegate updates pages → Navigator rebuilds UI.

Persisting Team with GetStorage

Objective

Save and restore the Pokémon team using GetStorage for persistent state management.

Setup


// pubspec.yaml
dependencies:
  get: ^4.6.6
  get_storage: ^2.1.1

// main.dart
void main() async {
  await GetStorage.init();
  runApp(MyApp());
}

Controller with GetStorage


class TeamController extends GetxController {
  var team = <String>[].obs;
  final storage = GetStorage();

  @override
  void onInit() {
    super.onInit();
    team.value = List<String>.from(storage.read('team') ?? []);
    ever(team, (_) => storage.write('team', team));
  }

  void addPokemon(String pokemon) {
    if (team.length >= 3) {
      Get.snackbar("Limit Reached", "Only 3 Pokémon allowed!");
    } else if (team.contains(pokemon)) {
      Get.snackbar("Duplicate", "$pokemon already in your team!");
    } else {
      team.add(pokemon);
    }
  }
}

Key Idea

  • ⚡ GetStorage.init() initializes persistent storage.
  • 📦 Automatically save team whenever it changes using ever().
  • 🔄 Restore saved team on app restart.

Dependency Injection with GetX

What is Dependency Injection?

A technique to provide objects/services to classes instead of creating them inside. GetX simplifies DI with Get.put(), Get.lazyPut(), and Get.find().

Example in Pokémon Team Builder


// Register Controller
void main() {
  Get.put(TeamController());       // Creates immediately
  Get.lazyPut(() => ApiService()); // Creates only when needed
  runApp(MyApp());
}

// Retrieve Controller
class TeamView extends StatelessWidget {
  final teamCtrl = Get.find<TeamController>();

  @override
  Widget build(BuildContext context) {
    return Obx(() => ListView(
      children: teamCtrl.team
          .map((pokemon) => Text(pokemon))
          .toList(),
    ));
  }
}

Key Methods

  • ⚡ Get.put() → creates and stores instance immediately.
  • 💤 Get.lazyPut() → creates only when first used.
  • 🔍 Get.find() → retrieves an existing instance.

Fetching Data with PokeAPI + GetX

Steps to Fetch Pokémon Data

  1. 📡 Create ApiService to fetch from https://pokeapi.co/api/v2/pokemon.
  2. 🗂️ Inject ApiService into TeamController with Get.lazyPut().
  3. 🔄 Store Pokémon list in an RxList and update UI with Obx.

Example Code


// Service to fetch Pokémon
class ApiService {
  final client = GetConnect();

  Future<List<String>> fetchPokemons() async {
    final response = await client.get("https://pokeapi.co/api/v2/pokemon?limit=20");
    if (response.statusCode == 200) {
      return (response.body['results'] as List)
        .map((p) => p['name'].toString())
        .toList();
    }
    return [];
  }
}

// TeamController
class TeamController extends GetxController {
  var pokemons = <String>[].obs;
  final ApiService api = Get.find();

  void loadPokemons() async {
    pokemons.value = await api.fetchPokemons();
  }
}

UI Binding


class PokemonList extends StatelessWidget {
  final teamCtrl = Get.find<TeamController>();

  @override
  Widget build(BuildContext context) {
    teamCtrl.loadPokemons();
    return Obx(() => ListView.builder(
      itemCount: teamCtrl.pokemons.length,
      itemBuilder: (ctx, i) => ListTile(
        title: Text(teamCtrl.pokemons[i]),
      ),
    ));
  }
}

Managing Team State with GetX

Objective

Allow user to select up to 3 Pokémon from the list and build their team.

Controller Update


class TeamController extends GetxController {
  var pokemons = <String>[].obs;
  var team = <String>[].obs;

  void togglePokemon(String name) {
    if (team.contains(name)) {
      team.remove(name);
    } else if (team.length < 3) {
      team.add(name);
    } else {
      Get.snackbar("Limit Reached", "You can only select 3 Pokémon!");
    }
  }
}

UI Example


class PokemonList extends StatelessWidget {
  final teamCtrl = Get.find<TeamController>();

  @override
  Widget build(BuildContext context) {
    return Obx(() => ListView.builder(
      itemCount: teamCtrl.pokemons.length,
      itemBuilder: (ctx, i) {
        final name = teamCtrl.pokemons[i];
        final selected = teamCtrl.team.contains(name);
        return ListTile(
          title: Text(name),
          trailing: Icon(
            selected ? Icons.check_circle : Icons.add_circle_outline,
            color: selected ? Colors.green : Colors.grey,
          ),
          onTap: () => teamCtrl.togglePokemon(name),
        );
      },
    ));
  }
}
