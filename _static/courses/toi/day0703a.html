ถัดมา (m) บรรทัด: (u) (v) หรือ (u) (v) (w)

for (int i = 0; i < m; i++) {

DSU Code Review
ทบทวนฟังก์ชัน DSU มาตรฐาน

// 1. find with path compression

int find(int x) { return link[x] == x ? x : link[x] = find(link[x]); }

// 2. unite with union by size

void unite(int a, int b) {

a = find(a); b = find(b);

if (a != b) {

if (size[a] < size[b]) swap(a, b);

size[a] += size[b]; link[b] = a;

}

}

Problem: Road Construction
โจทย์สร้างถนน (Connected Components)

เงื่อนไขโจทย์:

มีเมือง (n) เมือง และเริ่มสร้างถนนเชื่อมเมืองทีละเส้น

เป้าหมาย: ในทุกครั้งที่สร้างถนนเส้นใหม่ ให้ตอบว่า:

  • จำนวนกลุ่มเมือง (Components) ที่แยกจากกันในขณะนั้นคือเท่าใด?
  • ขนาดของกลุ่มเมืองที่ใหญ่ที่สุดคือเท่าใด?

แหล่งอ้างอิง: https://cses.fi/problemset/task/1676

Counting Strategy
กลยุทธ์การนับด้วย DSU

  • ● เริ่มแรกให้จำนวนกลุ่ม (count) เท่ากับเมืองทั้งหมด (n)
  • ● ทุกครั้งที่ทำ unite(u, v) สำเร็จ (u และ v เดิมอยู่คนละกลุ่ม)
  • ➜ ให้ลดค่า (count) ลง 1
  • สมการ: (count_{current} = n - successful\_merges)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Tracking Max Size
การหาขนาดกลุ่มที่ใหญ่ที่สุด

ใช้ตัวแปร `max_size` เพื่อเก็บค่าสูงสุดที่เคยเกิดขึ้น:

if (a != b) {
  unite(a, b);
  max_size = max(max_size, size[find(a)]);
}

"เมื่อกลุ่มสองกลุ่มรวมกัน ขนาดใหม่จะเป็นผลรวมของขนาดเดิมเสมอ"

Problem: Cycle Detection
การตรวจหาวงรอบในกราฟไม่ระบุทิศทาง

"ถนนเส้นนี้จำเป็นหรือไม่?"

  • ! ถ้าเมือง (u) และ (v) มี find(u) == find(v) อยู่แล้วก่อนเชื่อมถนน
  • ถนนเส้นนี้จะทำให้เกิด Cycle (วงรอบ)
  • i เหมาะสำหรับโจทย์ที่ถามว่ากราฟเป็น Tree หรือไม่ (Tree ต้องเชื่อมต่อกันและไม่มี Cycle)

แหล่งอ้างอิง: https://usaco.guide/CPH.pdf

Dynamic Connectivity
ความเชื่อมต่อที่มีการเปลี่ยนแปลง

DSU รองรับการ Add Edges อย่างมีประสิทธิภาพ (Additions only)

  • ถ้ามี Query ถามตลอดเวลาว่า (u) เชื่อมกับ (v) หรือไม่: ใช้ `find(u) == find(v)` ตอบได้ทันที
  • มีประสิทธิภาพดีกว่าการรัน DFS ใหม่ทุกครั้งที่รับ Input (ซึ่งใช้เวลา (O(Q \cdot (V+E))))
  • เวลาเฉลี่ยของ DSU คือ (O(Q \cdot \alpha(V))) ซึ่งรวดเร็วกว่ามหาศาล

แหล่งอ้างอิง: https://roadmap.sh/datastructures-and-algorithms

Implementation Gotchas
ข้อควรระวังในการเขียนกราฟและ DSU

1. Indexing

เช็คว่าโหนดเริ่มที่ 0 หรือ 1 (1-based vs 0-based) เสมอเพื่อจองอาเรย์ให้พอ

2. Self-loops

ถนนจากเมืองไปหาตัวเอง (u=v) ต้องข้ามไปหรือไม่? เช็คเงื่อนไขโจทย์ให้ดี

3. Multiple Edges

ถนนสองเส้นระหว่างเมืองคู่เดิม (Duplicate Edges) อาจไม่มีผลใน DSU แต่ส่งผลกับน้ำหนักใน MST

4. Memory

Adjacency List ประหยัดที่สุด แต่ถ้าโหนดเยอะอาเรย์ของ Vector อาจใช้แรมมหาศาล
