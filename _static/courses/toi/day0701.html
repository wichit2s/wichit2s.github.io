Graph Definition
นิยามและองค์ประกอบของกราฟ

กราฟ (G = (V, E)) ประกอบด้วยโหนดและเส้นเชื่อม

ส่วนประกอบหลัก

1
Nodes (Vertices - V): วัตถุหรือตำแหน่งในระบบ
2
Edges (E): เส้นเชื่อมระหว่างโหนด อาจมีทิศทางหรือไม่มีก็ได้
3
Weights: ค่าน้ำหนักกำกับเส้นเชื่อม เช่น ระยะทางหรือต้นทุน

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Graph Terminology
คำศัพท์ที่ต้องรู้จัก

พื้นฐานการท่องไปและการเชื่อมต่อในกราฟ

นิยามสำคัญ

1
Path: เส้นทางจากโหนดหนึ่งไปอีกโหนดหนึ่งผ่านเส้นเชื่อม
2
Cycle: เส้นทางที่จุดเริ่มต้นและจุดสิ้นสุดเป็นโหนดเดียวกัน
3
Connected: กราฟที่ทุกโหนดมีเส้นทางถึงกันได้

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Adjacency Matrix
การแทนกราฟด้วยเมทริกซ์

การใช้ Array 2 มิติ ขนาด (V \times V) เพื่อเก็บเส้นเชื่อม

คุณสมบัติ

1
ตรวจสอบการมีอยู่ของเส้นเชื่อมได้ในเวลา (O(1))
2
ใช้หน่วยความจำ (O(V^2)) ซึ่งอาจมากเกินไปสำหรับกราฟเบาบาง
3
เหมาะสำหรับกราฟหนาแน่น (Dense Graphs)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Adjacency List
การแทนกราฟด้วยรายการโหนดข้างเคียง

โครงสร้างที่นิยมที่สุด: `vector adj[N]`

ข้อดีและการใช้งาน

1
ประหยัดหน่วยความจำ ใช้เพียง (O(V+E))
2
ไล่ดูเพื่อนบ้านของโหนดได้รวดเร็ว เหมาะกับ BFS/DFS
3
เหมาะสำหรับกราฟเบาบาง (Sparse Graphs) ที่มีในโจทย์ส่วนใหญ่

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Disjoint Set Union (DSU)
โครงสร้างข้อมูลเซตที่แยกจากกัน

หรือเรียกอีกชื่อว่า Union-Find สำหรับจัดการกลุ่มข้อมูล

เป้าหมายหลัก

1
ตรวจสอบว่าโหนดสองโหนดอยู่ในกลุ่มเดียวกันหรือไม่
2
รวมกลุ่มโหนดสองกลุ่มเข้าด้วยกันอย่างรวดเร็ว
3
ใช้ในการตรวจสอบ Cycle และอัลกอริทึม MST

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Find and Unite
การดำเนินการหลักของ DSU

การจัดการกลุ่มผ่านตัวแทน (Representative)

กระบวนการ

1
Find(x): หาโหนดตัวแทนสูงสุดของกลุ่มที่ (x) สังกัดอยู่
2
Unite(a, b): นำโหนดตัวแทนของ (a) มาชี้ที่ตัวแทนของ (b) เพื่อรวมกลุ่ม
3
เริ่มแรกทุกโหนดเป็นตัวแทนของตัวเอง (Parent[i] = i)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Path Compression
การบีบอัดเส้นทางเพื่อความเร็ว

ลดความลึกของโครงสร้างต้นไม้ใน DSU

เทคนิค

1
ในขณะที่ทำ Find(x) ให้ทุกโหนดในเส้นทางชี้ตรงไปที่ตัวแทนกลุ่มทันที
2
เปลี่ยนความซับซ้อนจากเส้นตรงให้กลายเป็นเกือบค่าคงที่
3
`return parent[x] = find(parent[x]);`

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Union by Size
การรวมกลุ่มตามขนาดเพื่อความสมดุล

ป้องกันไม่ให้ต้นไม้มีความสูงมากเกินไป

กฎการรวม

1
ตรวจสอบขนาดของทั้งสองกลุ่มก่อนทำการรวม
2
นำกลุ่มที่มีขนาดเล็กกว่าไปต่อใต้กลุ่มที่มีขนาดใหญ่กว่าเสมอ
3
ช่วยควบคุมความลึกของต้นไม้ให้ไม่เกิน (O(\log N))

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Amortized Complexity
ประสิทธิภาพการทำงานที่เหนือชั้น

เมื่อใช้ทั้งสองเทคนิคเพิ่มประสิทธิภาพร่วมกัน

บทสรุปความเร็ว

1
ความซับซ้อนเฉลี่ยต่อการทำงานจะอยู่ที่ (O(\alpha(N)))
2
(\alpha(N)) คือ Inverse Ackermann function ซึ่งมีค่าน้อยมาก (< 5)
3
ในทางปฏิบัติถือว่าทำงานได้เป็นค่าคงที่ (Constant Time)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Adjacency List in C++
การเขียนโค้ดเพื่อเก็บข้อมูลกราฟ

ใช้ Vector Array เพื่อความยืดหยุ่นและประหยัดพื้นที่

// การประกาศโครงสร้างกราฟ

vector<int> adj[N];

// การเพิ่มเส้นเชื่อม (กรณี Undirected Graph)

adj[u].push_back(v);

adj[v].push_back(u);

// การเข้าถึงเพื่อนบ้านของโหนด x

for (auto b : adj[x]) { /* ประมวลผลโหนด b */ }

Adjacency Matrix in C++
การใช้ Array 2 มิติ

เหมาะสำหรับกราฟขนาดเล็ก (n < 5000)

// การประกาศเมทริกซ์

int adj[N][N];

// การเพิ่มเส้นเชื่อม

adj[a][b] = 1;

adj[b][a] = 1;

// การตรวจสอบว่า u และ v เชื่อมกันหรือไม่

if (adj[u][v]) { /* เชื่อมต่อกัน */ }

When to use which?
การเลือกโครงสร้างให้เหมาะกับงาน

Adjacency List

  • ใช้เมื่อโหนดเยอะแต่เส้นเชื่อมน้อย (Sparse)
  • พื้นที่: (O(V+E))
  • การท่องกราฟ: รวดเร็วมาก

Adjacency Matrix

  • ใช้เมื่อกราฟมีความหนาแน่นมาก (Dense)
  • พื้นที่: (O(V^2))
  • การลบ/ตรวจสอบเส้นเชื่อม: (O(1))

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

BFS & Queue Structure
การทำงานของ Breadth-First Search

ขั้นตอนการทำงาน

  1. ใช้ Queue เพื่อเก็บโหนดที่รอการประมวลผล
  2. ใช้ Array `visited[]` และ `distance[]` เพื่อป้องกันการวนซ้ำ
  3. โหนดใหม่จะถูกเพิ่มเข้าที่ท้าย Queue เสมอ
  4. ดึงโหนดจากหน้า Queue มาประมวลผลเพื่อนบ้านที่ยังไม่เคยไป

DSU: Find Operation
การหาตัวแทนกลุ่มด้วย Path Compression

// Find แบบมีประสิทธิภาพสูง

int find(int x) {

if (x == link[x]) return x;

return link[x] = find(link[x]); // Path Compression

}

// บรรทัดสีเขียวจะทำการปรับโครงสร้างต้นไม้ให้แบนลงในทุกครั้งที่เรียกใช้

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

DSU: Unite Operation
การรวมกลุ่มด้วย Union by Size

void unite(int a, int b) {

// หาตัวแทนของทั้งสองกลุ่ม

a = find(a); b = find(b);

if (a != b) {

// นำกลุ่มที่เล็กกว่าไปต่อกลุ่มที่ใหญ่กว่า

if (size[a] < size[b]) swap(a, b);

size[a] += size[b];

link[b] = a;

}

}

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Cycle Detection
การใช้ DSU ตรวจสอบวงรอบ

เทคนิคสำคัญสำหรับ Kruskal's Algorithm

เมื่อพิจารณาเส้นเชื่อมระหว่างโหนด (u) และ (v):

Case 1: ถ้า find(u) == find(v) แสดงว่าทั้งคู่เชื่อมกันอยู่แล้ว การเพิ่มเส้นเชื่อมนี้จะทำให้เกิด Cycle
Case 2: ถ้า find(u) != find(v) แสดงว่าอยู่คนละส่วน สามารถทำ unite(u, v) ได้อย่างปลอดภัย

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Dynamic Connectivity
การจัดการกราฟที่มีการเปลี่ยนแปลง

DSU สามารถจัดการการเพิ่มเส้นเชื่อมในระหว่างการประมวลผลได้

"โหนด (v) และ (w) เชื่อมต่อกันในปัจจุบันหรือไม่?"

  • ใช้แก้ปัญหาที่กราฟมีการเปลี่ยนแปลงตลอดเวลา (Additions only)
  • ทำงานได้เร็วกว่าการทำ BFS/DFS ใหม่ทุกครั้งที่เพิ่มเส้นเชื่อม
  • ประยุกต์ใช้ในปัญหาที่ต้องนับจำนวน Connected Components แบบ Real-time

แหล่งอ้างอิง: https://roadmap.sh/datastructures-and-algorithms

DSU เป็นหัวใจสำคัญของ Minimum Spanning Tree

  • รวมโหนดเข้าด้วยกันเพื่อสร้างต้นไม้ที่ครอบคลุมทุกโหนด

DFS Implementation
การเขียนโค้ดท่องกราฟแบบแนวลึก

void dfs(int s) {

if (visited[s]) return;

visited[s] = true;

// ประมวลผลโหนด s ที่นี่

for (auto u : adj[s]) {

dfs(u);

}

}

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Connectivity Check
การตรวจสอบการเชื่อมต่อของกราฟ

กราฟจะถือว่าเป็น **Connected** หากมีเส้นทางเชื่อมระหว่างทุกคู่โหนด:

  • เริ่มทำ DFS หรือ BFS จากโหนดใดโหนดหนึ่ง (เช่น โหนด 1)
  • หลังจากจบการทำงาน ให้ตรวจสอบ Array `visited[]`
  • หากมีโหนดใดที่ `visited` ยังเป็น `false` แสดงว่ากราฟไม่เชื่อมต่อกัน (Not Connected)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Connected Components
การนับจำนวนกลุ่มในกราฟ

int components = 0;

for (int i = 1; i <= n; i++) {

if (!visited[i]) {

components++;

dfs(i);

}

}

// จำนวนครั้งที่เริ่ม DFS ใหม่คือจำนวน Component ทั้งหมด

Bipartite Graphs
กราฟสองส่วน

คือโหนดในกราฟที่สามารถระบายสีได้ 2 สี โดยที่โหนดติดกันต้อง **คนละสี**:

  • ใช้ตรวจสอบว่ากราฟมี Cycle ที่เป็นจำนวนคี่หรือไม่
  • หากมี Odd Cycle กราฟนั้นจะ **ไม่เป็น** Bipartite
  • สามารถตรวจสอบได้โดยใช้ DFS/BFS และการระบายสีสลับกัน (2-Coloring)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Trees as Graphs
ความแตกต่างระหว่าง Tree และ Graph

"Tree is a special type of Graph"

Tree Properties

  • Connected
  • Acyclic (No Cycles)
  • (E = V - 1)
  • Exactly one path between any two nodes

General Graph

  • May be disconnected
  • May contain cycles
  • (E) can be up to (V(V-1)/2)
  • Multiple paths possible

DSU: Tracking Max Size
การติดตามขนาดกลุ่มที่ใหญ่ที่สุด

// ภายในฟังก์ชัน unite(a, b)

if (a != b) {

link[b] = a;

size[a] += size[b];

max_size = max(max_size, size[a]);

}

ช่วยให้ตอบคำถาม "กลุ่มที่ใหญ่ที่สุดในตอนนี้มีขนาดเท่าไหร่?" ได้ในเวลา (O(1))
