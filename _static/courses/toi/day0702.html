Spanning Tree
นิยามของต้นไม้เชื่อมโยง

ซับกราฟที่เป็นรากฐานของการเชื่อมต่อโหนดทั้งหมด

คุณสมบัติที่สำคัญ:

  • ✔ ประกอบด้วยโหนดทั้งหมดของกราฟเดิม
  • ✔ ต้องเชื่อมต่อกัน (Connected) และไม่มีวงรอบ (Acyclic)
  • ✔ ถ้ากราฟมี (n) โหนด ต้นไม้เชื่อมโยงจะมี (n-1) เส้นเชื่อมเสมอ

แหล่งอ้างอิง: https://roadmap.sh/datastructures-and-algorithms

Minimum Spanning Tree
ต้นไม้เชื่อมโยงที่น้อยที่สุด

MST คือต้นไม้เชื่อมโยงที่มี "น้ำหนักรวมของเส้นเชื่อม" น้อยที่สุดเท่าที่เป็นไปได้

• ค่าน้ำหนักรวมคำนวณจากผลรวมของค่าน้ำหนักบนเส้นเชื่อมในต้นไม้

• กราฟหนึ่งอาจมี MST ได้มากกว่าหนึ่งรูปแบบ แต่จะมีน้ำหนักรวมเท่ากันเสมอ

• ใช้แก้ปัญหาทางวิศวกรรม เช่น การวางสายเคเบิลหรือถนนด้วยงบประมาณต่ำสุด

แหล่งอ้างอิง: https://www.geeksforgeeks.org/greedy-algorithms/

Kruskal’s Algorithm
กลยุทธ์แบบ Greedy ของ Kruskal

"เลือกเส้นเชื่อมที่มีค่าน้ำหนักน้อยที่สุดก่อนเสมอ"

แนวคิดหลัก

เริ่มต้นจากโหนดที่แยกกัน แล้วค่อยๆ นำเส้นเชื่อมที่สั้นที่สุดมาต่อเข้าด้วยกันโดยไม่ให้เกิด Cycle

โครงสร้างที่ใช้

1. Edge List (สำหรับการ Sort)
2. Disjoint Set Union (สำหรับตรวจสอบ Cycle)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Step-by-Step Kruskal
ลำดับขั้นตอนการทำงาน

  1. นำเส้นเชื่อม (Edges) ทั้งหมดมา Sort ตามน้ำหนักจากน้อยไปมาก
  2. วนลูปพิจารณาเส้นเชื่อมทีละเส้นตามลำดับน้ำหนัก
  3. ใช้ฟังก์ชัน sameSet(u, v) ของ DSU ตรวจสอบว่าโหนด u และ v อยู่ในกลุ่มเดียวกันหรือไม่
  4. ถ้าไม่อยู่กลุ่มเดียวกัน ให้เลือกเส้นนั้นเข้า MST และเรียก unite(u, v)
  5. ทำจนกว่าจะได้เส้นเชื่อม (n-1) เส้น

แหล่งอ้างอิง: https://www.geeksforgeeks.org/kruskals-minimum-spanning-tree-algorithm-greedy-algo-2/

The Power of DSU
ทำไมต้องใช้ DSU ใน Kruskal?

// Pseudocode logic

sort(edges.begin(), edges.end());

for (auto e : edges) {

if (!dsu.sameSet(e.u, e.v)) {

// u และ v ยังไม่เชื่อมกัน ไม่เกิด Cycle แน่นอน

dsu.unite(e.u, e.v);

mst_weight += e.w;

}

}

ความซับซ้อน: (O(E \log E)) เนื่องจากการ Sort เส้นเชื่อมเป็นหลัก

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Prim’s Algorithm
อัลกอริทึมของพริม: การเติบโตของต้นไม้

ขยายขอบเขตจากโหนดเริ่มต้นไปยังโหนดข้างเคียงที่ใกล้ที่สุด

จุดเด่น:

  • เริ่มจากโหนดใดก็ได้เพียงหนึ่งโหนด
  • ในแต่ละก้าว จะเลือกเส้นเชื่อมที่มีน้ำหนักน้อยที่สุดที่เชื่อมระหว่าง "โหนดในต้นไม้" กับ "โหนดนอกต้นไม้"
  • ใช้ Priority Queue (Min-Heap) ในการเก็บและดึงเส้นเชื่อมที่สั้นที่สุดอย่างมีประสิทธิภาพ

แหล่งอ้างอิง: https://www.geeksforgeeks.org/prims-minimum-spanning-tree-mst-greedy-algo-5/

Prim’s Algorithm Implementation
ขั้นตอนการทำงานด้วย Min-Heap

1. เลือกโหนดเริ่มต้นและตั้งระยะห่างของโหนดอื่นเป็น ( \infty )
2. ใส่โหนดเริ่มต้นลงใน Priority Queue ด้วยระยะทาง 0
3. ดึงโหนด (u) ที่มีระยะทางน้อยสุดออกจาก Queue
4. วนดูเพื่อนบ้าน (v) ของ (u): หากเส้นเชื่อม (u-v) สั้นกว่าระยะทางเดิมของ (v) ให้ปรับปรุงค่าและใส่ (v) ลงใน Queue
5. ทำซ้ำจนกว่าโหนดจะถูกเลือกเข้า MST ครบทุกโหนด

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Kruskal vs Prim
เลือกใช้อะไรดี?

Kruskal

  • ทำงานได้ดีในกราฟที่มีเส้นเชื่อมน้อย (Sparse Graphs)
  • ง่ายต่อการ Implement ด้วย DSU
  • นิยมมากใน Competitive Programming

Prim

  • ทำงานได้รวดเร็วในกราฟที่มีเส้นเชื่อมหนาแน่น (Dense Graphs)
  • ความซับซ้อน (O(E \log V))
  • โครงสร้างคล้ายกับ Dijkstra

"Competitive Programmer ส่วนใหญ่เลือก Kruskal เพราะ Edge List จัดการง่ายกว่า"

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Introduction to DAGs
กราฟระบุทิศทางที่ไม่มีวงรอบ

สะพานเชื่อมสู่การจัดเรียงลำดับความสำคัญ

DAG (Directed Acyclic Graph) คือกราฟที่มีทิศทางและ "ไม่มี Cycle":

  • ใช้แทนความสัมพันธ์เชิงลำดับ เช่น วิชาที่ต้องเรียนก่อน-หลัง
  • เป็นโครงสร้างเดียวที่สามารถทำ Topological Sort ได้
  • เป็นพื้นฐานของการทำ Dynamic Programming บนกราฟ (DP on DAGs)

แหล่งอ้างอิง: https://usaco.guide/CPH.pdf

Kahn’s Algorithm
การจัดเรียงด้วย Breadth-First Search

หลักการของ In-degree

  • In-degree: จำนวนเส้นเชื่อมที่ชี้เข้าหาโหนดนั้น ๆ
  • 1. คำนวณ In-degree ของทุกโหนดในกราฟ
  • 2. นำโหนดที่มี (In-degree = 0) ใส่ลงใน Queue (เพราะไม่มีโหนดที่ต้องทำก่อน)
  • 3. ดึงโหนดออกมาระบุลำดับ และ "ลดค่า" In-degree ของเพื่อนบ้านลง
  • 4. ถ้าเพื่อนบ้านตัวไหนกลายเป็น 0 ให้นำเข้า Queue ต่อไป

แหล่งอ้างอิง: https://www.geeksforgeeks.org/topological-sorting-indegree-based-solution/

DFS for Topo Sort
การใช้ความลึกในการจัดลำดับ

ต่างจาก BFS ตรงที่เราจะสำรวจไปให้ลึกที่สุดก่อน:

  1. เริ่มทำ DFS จากโหนดที่ยังไม่เคยไป
  2. เดินไปจนสุดทาง (ไม่สามารถไปต่อได้)
  3. เมื่อโหนดนั้น "ประมวลผลเสร็จ" (Post-order) ให้เก็บเข้า Stack หรือ List
  4. ลำดับที่ถูกต้องคือการ Reverse รายการโหนดที่เราเก็บมา

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Cycle Detection
การตรวจหาวงรอบในกราฟระบุทิศทาง

กลยุทธ์ 3 สถานะ (3-Coloring)

State 0: ยังไม่เคยเยี่ยมชม (Unvisited)
State 1: กำลังประมวลผล (Visiting)
State 2: ประมวลผลเสร็จแล้ว (Visited)

"หาก DFS ไปเจอโหนดที่อยู่ในสถานะ Visiting ซ้ำ แสดงว่ามี Cycle!"

แหล่งอ้างอิง: https://roadmap.sh/datastructures-and-algorithms

DP on DAG
หัวใจของการประมวลผลตามลำดับ

เมื่อกราฟไม่มี Cycle เราสามารถใช้ลำดับจากการทำ Topological Sort เพื่อทำ DP ได้อย่างมีประสิทธิภาพ:

  • การหาเส้นทางสั้นที่สุด/ยาวที่สุดใน DAG
  • การนับจำนวนเส้นทางจากโหนดเริ่มต้นไปยังเป้าหมาย
  • หลักการ: คำนวณค่าของโหนด (u) ก่อนโหนด (v) เสมอถ้ามีเส้นเชื่อม (u \to v)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Path Counting
การนับจำนวนทางเดินใน DAG

(paths[v] = \sum_{u \to v} paths[u])

1. เริ่มให้ (paths[start] = 1)

2. ไล่ตามลำดับ Topological Sort

3. สำหรับแต่ละโหนด (u) ให้ส่งค่าไปบวกเพิ่มให้เพื่อนบ้าน (v) ทุกตัว

แหล่งอ้างอิง: https://usaco.guide/CPH.pdf

Longest Path in DAG
การหาเส้นทางที่ยาวที่สุด

ปกติการหา Longest Path ในกราฟทั่วไปเป็นปัญหาที่ยากมาก (NP-Hard) แต่ใน DAG เราสามารถใช้ DP ช่วยได้:

(dist[v] = \max(dist[v], dist[u] + weight(u, v)))

ใช้ในปัญหาเช่น การหาเวลาที่เร็วที่สุดที่จะเริ่มงานสุดท้ายได้ หลังจากงานก่อนหน้าทั้งหมดเสร็จสิ้น (Critical Path Method)

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Dijkstra’s Strategy
กลยุทธ์แบบ Greedy ของไดก์สตรา

1. เริ่มต้นให้ระยะทางของโหนดเริ่มเป็น 0 โหนดอื่นเป็น (\infty)

2. ใช้ Priority Queue (Min-Heap) เก็บโหนดที่พร้อมจะ "ผ่อนคลาย" (Relax)

3. เลือกโหนดที่ใกล้ที่สุดออกมาเสมอ

4. Edge Relaxation: หากเดินผ่านโหนดปัจจุบันไปยังโหนดถัดไปแล้วได้ระยะทางที่ "สั้นกว่าเดิม" ให้ปรับปรุงค่า

Bellman-Ford Algorithm
การจัดการน้ำหนักติดลบ

อัลกอริทึมนี้จะทำ Relaxation เส้นเชื่อม "ทั้งหมด" จำนวน (n-1) รอบ:

  • ทำไมต้อง (n-1) รอบ? เพราะเส้นทางสั้นที่สุดที่ไม่มีวงรอบจะมีเส้นเชื่อมไม่เกิน (n-1) เส้น
  • Negative Cycle Detection: หากทำรอบที่ (n) แล้วระยะทางยัง "ลดลง" ได้อีก แสดงว่ามีวงรอบติดลบในกราฟ!
  • ความซับซ้อน: (O(V \cdot E)) ช้ากว่า Dijkstra แต่ใช้ในกรณีพิเศษได้ดี

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Floyd-Warshall
สั้นที่สุดระหว่างทุกคู่โหนด

for (int k = 1; k <= n; k++)

for (int i = 1; i <= n; i++)

for (int j = 1; j <= n; j++)

adj[i][j] = min(adj[i][j], adj[i][k] + adj[k][j]);

• ใช้เมทริกซ์ประชิด (Adjacency Matrix) ในการเก็บข้อมูล

• ความซับซ้อน (O(V^3)) เหมาะกับกราฟขนาดเล็ก ((n \le 500))

แหล่งอ้างอิง: https://www.geeksforgeeks.org/floyd-warshall-algorithm-dp-16/

Implementation Kruskal
เคล็ดลับการเขียนโค้ด Kruskal

  • • ใช้ struct หรือ tuple เก็บ (weight, u, v) เพื่อให้ใช้ `sort()` ของ STL ได้ทันที
  • • อย่าลืมฟังก์ชัน `iota()` ในการเริ่มต้น Parent array ของ DSU
  • • ตรวจสอบ Cycle ด้วย `find(u) == find(v)` ก่อนสั่ง `unite()`
  • • การ Sort เส้นเชื่อมใช้เวลา (O(E \log E)) ซึ่งเป็นส่วนที่ใช้เวลามากที่สุด

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Implementation Prim
เคล็ดลับการเขียนโค้ด Prim

  • • ใช้ `priority_queue>` โดยใส่ค่าน้ำหนักเป็น ค่าติดลบ เพื่อจำลอง Min-Heap
  • • ใช้ array `visited[]` เพื่อระบุว่าโหนดนั้นอยู่ใน MST แล้วหรือยัง
  • • หากโหนดที่ดึงมาจาก Queue ถูกเยี่ยมชมแล้ว ให้ `continue` ข้ามไปทันที
  • • ความซับซ้อน (O(E \log V)) เหมาะกับกราฟที่มีความหนาแน่น

แหล่งอ้างอิง: https://cses.fi/book/book.pdf

Graph Hierarchy
ลำดับขั้นของโครงสร้างกราฟ

  • Tree: เชื่อมต่อกัน, ไม่มี Cycle, (E = V-1)
  • DAG: มีทิศทาง, ไม่มี Cycle, ใช้ทำ Topological Sort ได้
  • General Graph: มี Cycle ได้, อาจไม่เชื่อมต่อ, ต้องใช้ระวังเรื่อง Infinite Loop

แหล่งอ้างอิง: https://usaco.guide/CPH.pdf

Edge Relaxation
แนวคิดพื้นฐานการหาทางลัด

"หากเรารู้ระยะทางไป (u) และมีทางเดิน (u \to v) ที่สั้นกว่าทางเดิมที่เคยรู้ไป (v) จงเปลี่ยนไปใช้ทางนั้นซะ!"

if (d[u] + w < d[v]) d[v] = d[u] + w;

เป็นหัวใจของ Dijkstra, Bellman-Ford และ Floyd-Warshall

Prim’s Implementation
การใช้ Priority Queue ในอัลกอริทึมของพริม

priority_queue<pair<int,int>> q;

q.push({0, 1}); // {weight, node}

while (!q.empty()) {

int a = q.top().second; q.pop();

if (visited[a]) continue;

visited[a] = true;

for (auto u : adj[a]) {

if (u.weight < dist[u.node]) {

dist[u.node] = u.weight;

q.push({-dist[u.node], u.node});

}

}

}

Topological Sort Code
การเขียนโค้ดจัดเรียงลำดับ

void dfs(int s) {

state[s] = 1; // อยู่ระหว่างประมวลผล

for (auto u : adj[s]) {

if (state[u] == 0) dfs(u);

}

state[s] = 2; // ประมวลผลเสร็จแล้ว

result.push_back(s);

}

Cycle Detection
การตรวจหาวงรอบในกราฟระบุทิศทาง

หลักการ 3 สถานะ:

  • 0 Unvisited: ยังไม่เคยเข้าไปเยี่ยมชม
  • 1 Visiting: กำลังทำงานอยู่ (ยังไม่หลุดออกจาก Recursion)
  • 2 Visited: ประมวลผลโหนดนั้นและลูกหลานเสร็จสิ้นแล้ว

หาก DFS ไปเจอโหนดที่อยู่ในสถานะ 1 แสดงว่ากราฟมี Cycle!

Longest Path in DAG
การหาเส้นทางที่ยาวที่สุดด้วย DP

ในกราฟทั่วไปการหา Longest Path เป็น NP-Hard แต่ใน **DAG** สามารถทำได้ใน (O(V+E))

for (int u : topo_order) {
  for (auto v : adj[u]) {
    dist[v] = max(dist[v], dist[u] + weight(u,v));
  }
}

Dijkstra’s Algorithm
อัลกอริทึมของไดก์สตรา

  • เป็น **Greedy Algorithm** ที่ใช้หาเส้นทางสั้นที่สุดจากจุดเดียว (Single-source)
  • ใช้ได้กับกราฟที่มีน้ำหนักเป็นบวกเท่านั้น (Non-negative weights)
  • ทำงานคล้ายกับ Prim’s Algorithm แต่เปลี่ยนจากการเก็บระยะทางไปยังต้นไม้ เป็นระยะทางไปยังจุดเริ่มต้น
  • ความซับซ้อน: (O(V + E \log E)) หรือ (O(V + E \log V))

แหล่งอ้างอิง: https://www.geeksforgeeks.org/dijkstras-shortest-path-algorithm-greedy-algo-7/

Bellman-Ford Algorithm
การผ่อนคลายเส้นเชื่อม (Edge Relaxation)

  • ใช้หา Single-source shortest path แม้จะมีน้ำหนักติดลบ
  • หลักการ: ทำการ **Relax** เส้นเชื่อมทั้งหมด (E) เป็นจำนวน (n-1) รอบ
  • หลังจาก (n-1) รอบ ระยะทางที่ได้จะเป็นค่าที่สั้นที่สุด
  • หากทำรอบที่ (n) แล้วค่าระยะทางยังลดลงได้อีก แสดงว่ากราฟมี **Negative Cycle**
  • ความซับซ้อน: (O(VE)) ช้ากว่า Dijkstra แต่ยืดหยุ่นกว่า

แหล่งอ้างอิง: https://cses.fi/book/book.pdf
