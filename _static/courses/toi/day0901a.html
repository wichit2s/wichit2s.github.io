Greedy vs Divide & Conquer

Greedy

  • เลือกคำตอบที่ดีที่สุดในแต่ละขั้น
  • ไม่ย้อนกลับ
  • ง่ายและเร็ว
  • ใช้ได้เฉพาะบางปัญหา
Example: Activity Selection

Divide & Conquer

  • แบ่งปัญหาเป็นส่วนย่อย
  • แก้แต่ละส่วนแบบ recursive
  • รวมผลลัพธ์
Example: Merge Sort, Binary Search

Why Greedy Sometimes Fails?

Greedy ใช้ได้เฉพาะปัญหาที่มีคุณสมบัติ

1
Greedy Choice Property
2
Optimal Substructure

Greedy Algorithms

แนวคิดการเลือกคำตอบที่ดีที่สุดในแต่ละขั้นตอน
1
เลือกทางเลือกที่ดีที่สุดในปัจจุบัน
2
ไม่ย้อนกลับไปแก้คำตอบก่อนหน้า
3
หวังว่าการเลือกที่ดีที่สุดในแต่ละขั้นจะนำไปสู่คำตอบที่ดีที่สุดโดยรวม
Local Optimal → Global Optimal

Greedy Principle

Greedy Algorithm ทำงานโดยการเลือกคำตอบที่ดีที่สุดในแต่ละขั้นตอน

Local Optimal
คำตอบที่ดีที่สุดในขั้นตอนปัจจุบัน
Global Optimal
คำตอบที่ดีที่สุดของทั้งปัญหา

หากปัญหามีคุณสมบัติ Greedy Choice Property

Local Optimal → Global Optimal

Greedy Choice Property

ปัญหาที่ใช้ Greedy ได้ต้องมีคุณสมบัติสำคัญ 2 อย่าง

1
Greedy Choice Property สามารถเลือกทางเลือกที่ดีที่สุดในตอนนี้ได้เลย
2
Optimal Substructure ปัญหาสามารถแบ่งเป็นปัญหาย่อยที่เหมือนกันได้
หากไม่มีคุณสมบัตินี้ Greedy จะให้คำตอบผิด

Merge Sort (C++) 

void mergeSort(vector& a,int l,int r){
   if(l>=r) return;
   int mid=(l+r)/2;
   mergeSort(a,l,mid);
   mergeSort(a,mid+1,r);
   merge(a,l,mid,r);
}

Merge Sort Complexity

T(n) = 2T(n/2) + O(n)
Time Complexity = O(n log n)
Depth of recursion tree = log n Work per level = n

Quick Sort (C++) 

int partition(vector<int>& a,int l,int r){
   int pivot=a[r];
   int i=l-1;
   for(int j=l;j<r;j++) {
      if(a[j]<pivot){
         i++;
         swap(a[i],a[j]);
      }
   }
   swap(a[i+1],a[r]);
   return i+1;
}

Quick Sort Complexity

Best Case O(n log n)
Average Case O(n log n)
Worst Case O(n²)

Merge Sort vs Quick Sort

Merge Sort

  • Stable
  • O(n log n)
  • Extra memory required

Quick Sort

  • In-place
  • Usually faster
  • Worst case O(n²)
