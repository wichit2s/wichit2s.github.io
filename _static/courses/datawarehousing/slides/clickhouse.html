ความแตกต่างระหว่าง
OLAP และ OLTP

เปรียบเทียบการใช้งานและลักษณะของระบบทั้งสองประเภท

OLAP (Online Analytical Processing)

  • เน้นการวิเคราะห์ข้อมูลและรายงาน
  • ข้อมูลในอดีตและสรุปผล
  • คำสั่งอ่านข้อมูลจำนวนมาก (Read-heavy)
  • เหมาะกับการสอบถามข้อมูลเชิงลึกและการทำ Business Intelligence
  • โครงสร้างข้อมูลมักเป็น Star Schema หรือ Snowflake Schema
  • การอัปเดตข้อมูลแบบเป็นชุด (Batch)

OLTP (Online Transaction Processing)

  • เน้นการจัดการธุรกรรมประจำวัน
  • ข้อมูลแบบเรียลไทม์และปัจจุบัน
  • คำสั่งเขียนและอ่านแบบเร็ว (Write-heavy)
  • เหมาะกับการทำงานแบบ transactional เช่น ระบบธนาคาร ระบบจอง
  • โครงสร้างข้อมูลแบบ normalized เพื่อลดความซ้ำซ้อน
  • อัปเดตข้อมูลอย่างต่อเนื่องและถี่

จุดเด่นของ ClickHouse ในการรองรับงานที่มีการเขียนข้อมูลความเร็วสูง
  • ✔ จัดเก็บข้อมูลแบบ Columnar เหมาะกับการอ่านและวิเคราะห์ข้อมูลเร็ว
  • ✔ รองรับการ Insert ข้อมูลจำนวนมากแบบ Batch ได้ดี
  • ✔ สามารถใช้ MergeTree และ ReplacingMergeTree สำหรับ Deduplication
  • ✔ เหมาะกับระบบเก็บ Event Log, Metrics และ Clickstream

เปรียบเทียบ
MergeTree vs ReplacingMergeTree

ความแตกต่างและการเลือกใช้งาน Engine ใน ClickHouse

MergeTree

  • Engine มาตรฐานของ ClickHouse
  • รองรับการจัดเรียงข้อมูลตาม ORDER BY
  • เหมาะสำหรับการเก็บข้อมูล Append-only
  • ไม่จัดการข้อมูลซ้ำซ้อนโดยอัตโนมัติ
  • เหมาะกับการเก็บ Log หรือ Event จำนวนมาก

ReplacingMergeTree

  • สืบทอดจาก MergeTree
  • รองรับการแทนที่ (Deduplication) ระหว่างการ Merge
  • ใช้ primary key หรือ version column กำหนดข้อมูลล่าสุด
  • เหมาะสำหรับกรณีต้องมีการ Update ข้อมูล
  • ใช้กับกรณี Upsert เช่น สถานะล่าสุดของผู้ใช้

แนวทางปฏิบัติที่ดีที่สุด
สำหรับใช้งาน ClickHouse OLTP

เคล็ดลับและเทคนิคเพื่อเพิ่มประสิทธิภาพและความเสถียรในการใช้งาน
  • หลีกเลี่ยงการแก้ไขหรืออัปเดตข้อมูลแบบขนาดใหญ่ ใช้วิธี insert ทับด้วย ReplacingMergeTree แทน
  • ออกแบบ ORDER BY และ PARTITION BY ให้เหมาะกับรูปแบบการ query และปริมาณข้อมูล
  • ใช้ Buffer Tables เพื่อรองรับโหลดการ insert ข้อมูลจำนวนมากอย่างต่อเนื่อง
  • ตั้งค่า TTL สำหรับลบข้อมูลเก่าที่ไม่จำเป็น เพื่อลดขนาดฐานข้อมูล
  • จัดการการทำ deduplication ด้วย ReplacingMergeTree เพื่อป้องกันข้อมูลซ้ำซ้อน
  • ตรวจสอบและปรับแต่ง config ของระบบ เช่น max_insert_block_size และ merge settings เพื่อเพิ่มประสิทธิภาพ

ขั้นตอนที่ 2
กำหนด ASGI และ Routing

ตั้งค่าไฟล์ settings.py และสร้าง routing.py สำหรับ WebSocket 
              # settings.py
              ASGI_APPLICATION = 'myproject.asgi.application'

              # asgi.py
              application = ProtocolTypeRouter({
              "websocket": AuthMiddlewareStack(
              URLRouter(
              myapp.routing.websocket_urlpatterns
              )
              ),
              })

              # routing.py
              from django.urls import re_path
              from . import consumers

              websocket_urlpatterns = [
              re_path(r'ws/events/$', consumers.EventConsumer.as_asgi()),
              ]

ขั้นตอนที่ 3
สร้างไฟล์ Consumer

เขียน Consumer สำหรับจัดการ WebSocket และเชื่อมต่อ ClickHouse 
              # consumers.py
              from channels.generic.websocket import AsyncWebsocketConsumer
              import json
              from clickhouse_connect import get_client

              class EventConsumer(AsyncWebsocketConsumer):
              async def connect(self):
              await self.accept()

              async def receive(self, text_data):
              client = get_client(host='localhost')
              result = client.query('SELECT count() FROM events')
              await self.send(text_data=json.dumps({'count': result.result_rows[0][0]}))

ขั้นตอนที่ 4
เชื่อมต่อ ClickHouse

สร้างฟังก์ชันหรือ Utility สำหรับจัดการการเชื่อมต่อ ClickHouse ในโปรเจกต์ Django 
              # clickhouse_client.py
              from clickhouse_connect import get_client

              client = get_client(host='localhost', username='default', password='')

              def query_clickhouse(sql):
              result = client.query(sql)
              return result.result_rows

ขั้นตอนที่ 5
Frontend ด้วย Alpine.js และ HTMX

เชื่อม WebSocket ด้วย Alpine.js และอัปเดตข้อมูลแบบ Realtime ด้วย HTMX

ตัวอย่างโค้ด HTML + Alpine.js + HTMX สำหรับรับข้อมูลแบบเรียลไทม์

              <div
              x-data=\"{ count: 0 }\"
              x-init=\"(() => {
              const ws = new WebSocket('ws://' + window.location.host + '/ws/events/');
              ws.onmessage = event => {
              const data = JSON.parse(event.data);
              count = data.count;
              };
              })()\"
              >
              <p>จำนวนเหตุการณ์ล่าสุด: <span x-text=\"count\">0</span></p>
              </div>

เพิ่ม script สำหรับ Alpine.js และ HTMX ใน <head> หรือ ก่อนปิด </body>:

              <script src=\"https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js\" defer></script>
              <script src=\"https://unpkg.com/htmx.org@1.9.2\"></script>
