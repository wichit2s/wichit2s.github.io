ทำไมต้องใช้
ClickHouse กับ OLTP

จุดเด่นของ ClickHouse ในการรองรับงานที่มีการเขียนข้อมูลความเร็วสูง
  • ✔ จัดเก็บข้อมูลแบบ Columnar เหมาะกับการอ่านและวิเคราะห์ข้อมูลเร็ว
  • ✔ รองรับการ Insert ข้อมูลจำนวนมากแบบ Batch ได้ดี
  • ✔ สามารถใช้ MergeTree และ ReplacingMergeTree สำหรับ Deduplication
  • ✔ เหมาะกับระบบเก็บ Event Log, Metrics และ Clickstream
เหตุผลหลัก

สถาปัตยกรรม
ClickHouse

ส่วนประกอบหลักและวิธีทำงานภายใน ClickHouse
  • ✔ Storage Engine: MergeTree, ReplacingMergeTree
  • ✔ Distributed Processing: การกระจายการอ่าน-เขียน
  • ✔ Columnar Storage: เก็บข้อมูลแบบคอลัมน์เพื่อความเร็ว
  • ✔ การจัดการ Index และการจัดเรียงข้อมูล

การติดตั้ง
ClickHouse

วิธีติดตั้งและตั้งค่า ClickHouse เบื้องต้น
  • ✔ ติดตั้งผ่าน Docker: ใช้ image มาตรฐานของ ClickHouse
  • ✔ ติดตั้ง Binary บน Linux หรือ MacOS
  • ✔ เชื่อมต่อด้วย clickhouse-client
  • ✔ ไฟล์ config สำคัญ: config.xml, users.xml

การสร้างตาราง
สำหรับ OLTP

โครงสร้างตารางและ Engine ที่เหมาะกับงาน OLTP-like ใน ClickHouse
  • ✔ ใช้ MergeTree เป็น Engine พื้นฐาน
  • ✔ ReplacingMergeTree สำหรับการจัดการข้อมูลซ้ำ
  • ✔ ORDER BY เพื่อประสิทธิภาพการอ่านข้อมูล
  • ✔ PARTITION BY ช่วยจัดการข้อมูลขนาดใหญ่
ออกแบบ Schema

การ Insert และ Query
ข้อมูลใน ClickHouse

เทคนิคการแทรกข้อมูลจำนวนมากและดึงข้อมูลได้รวดเร็ว
  • ✔ ใช้ INSERT แบบ Batch เพื่อประสิทธิภาพสูงสุด
  • ✔ ออกแบบ ORDER BY ให้เหมาะกับ Query Pattern
  • ✔ ใช้ LIMIT, SAMPLE, และ INDEX เพื่อเร่งความเร็ว
  • ✔ หลีกเลี่ยง UPDATE ขนาดใหญ่ ใช้การ Insert ทับแทน

ClickHouse OLTP Workshop

ประเด็นสำคัญและโอกาสในการนำ ClickHouse ไปใช้กับงาน OLTP-like
  • ✔ ClickHouse เหมาะกับงานที่เน้น Insert & Query สูง แต่ไม่ซับซ้อนด้าน Transaction
  • ✔ การออกแบบ Schema และการเลือก Engine มีผลต่อประสิทธิภาพมาก
  • ✔ ควรใช้ Buffer Tables และ TTL เพื่อจัดการข้อมูลแบบเรียลไทม์อย่างเหมาะสม
