The Foundation: Class vs. Object

Class (The Blueprint)

  • A logical definition or template.
  • Defines attributes (data) and methods (behavior).
  • Does not occupy memory until instantiated.
  • Example: `class Dog:`

Object (The Instance)

  • A concrete entity created from a class.
  • Has a distinct state (data values) and identity.
  • Occupies memory at runtime.
  • Example: `my_dog = Dog("Buddy")`

The 4 Pillars of OOP

1. Encapsulation

Bundling data and methods together and restricting direct access to object components (e.g., private variables).

2. Inheritance

Creating new classes based on existing ones to promote code reuse (The "Is-A" relationship).

3. Polymorphism

The ability of different objects to respond to the same method call in different ways (e.g., Method Overriding).

4. Abstraction

Hiding complex implementation details and showing only the essential features of the object (e.g., Abstract Classes).

Bundling data and methods into a single unit and restricting access to the object's internal state.

Access Modifiers in Python

Pillar 2: Inheritance

Code Reuse ("Is-A")

A child class derives attributes and methods from a parent class. It can extend or override functionality.

The super() Function

Returns a proxy object that delegates method calls to a parent or sibling class. Essential for accessing inherited methods. 

class Animal:
    def speak(self):
        print("Generic sound")

class Dog(Animal):  # Inheritance
    def speak(self):
        # Override parent method
        print("Woof!")

class SuperDog(Dog):
    def speak(self):
        # Call parent implementation
        super().speak() 
        print("...and I can fly!")

Multiple Inheritance & MRO

Python supports inheriting from multiple classes. Order matters to avoid the "Diamond Problem."

Method Resolution Order (MRO)

The linearization algorithm (C3) Python uses to determine the order in which to search for methods.

>>> ClassName.__mro__
(Child, Parent1, Parent2, object)

Python searches left-to-right, depth-first (mostly).

class A:
    def do_task(self): print("A")

class B(A): pass

class C(A):
    def do_task(self): print("C")

# Multiple Inheritance
class D(B, C): pass

d = D()
d.do_task()  
# Output: "C" 
# MRO: D -> B -> C -> A

Pillar 4: Abstraction

Hiding Complexity

Abstraction allows you to define a template (interface) while hiding the complex implementation details. It forces subclasses to adhere to a specific contract.

  • ABC: Abstract Base Classes (from abc module).
  • @abstractmethod: Methods that must be implemented by children.
  • Enforcement: Python throws an error if you try to instantiate an incomplete abstract class.
from abc import ABC, abstractmethod

class PaymentGateway(ABC):
    @abstractmethod
    def process(self, amount):
        pass

class PayPal(PaymentGateway):
    def process(self, amount):
        print(f"Paid ${amount} via PayPal")

# pg = PaymentGateway() # Error!
p = PayPal()
p.process(100) # Output: Paid $100 via PayPal

Composition over Inheritance

Inheritance (Is-A)

Use when the child is a specialized version of the parent.

class Manager(Employee):
  ...

Rigid relationship. Changes in parent affect all children.

Composition (Has-A)

Use to build complex objects from smaller, independent components.

class Car:
  def __init__(self):
    self.engine = Engine()

Flexible. Components can be swapped at runtime.

Method Types: Instance vs. Class vs. Static

Instance Methods

The default method type. Accesses and modifies object state.

  • First Arg: self
  • Access: Instance & Class
def work(self):
  self.data = 1

Class Methods

Bound to the class, not the instance. Often used as Factory Methods.

  • Decorator: @classmethod
  • First Arg: cls
@classmethod
def from_string(cls, s):
  return cls(s)

Static Methods

Utility functions logically related to the class but independent of state.

  • Decorator: @staticmethod
  • First Arg: None
@staticmethod
def is_valid(x):
  return x > 0

Managed Attributes with @property

Don't write Java-style get_variable() and set_variable() methods. Use the Property decorator to validate data or compute values dynamically while keeping a clean interface.

Benefits

  • Validation: Check values before assignment (e.g., age cannot be negative).
  • Computed Fields: Calculate values on the fly (e.g., area from width * height).
  • Backward Compatibility: Change implementation without changing the public API.
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def diameter(self):
        # Computed attribute
        return self._radius * 2

    @diameter.setter
    def diameter(self, value):
        # Validation logic
        if value < 0:
            raise ValueError
        self._radius = value / 2

Python Magic Methods (__dunder__)

Define how objects behave with built-in Python operators and functions.

String Representation

  • __str__(self) For end-users (print)
  • __repr__(self) For developers (debugging)

Operator Overloading

  • __add__(self, other) obj1 + obj2
  • __eq__(self, other) obj1 == obj2

Containers

  • __len__(self) len(obj)
  • __getitem__(self, key) obj[key]

Context Managers

  • __enter__(self) with obj:
  • __exit__(self, ...) cleanup logic

Adding the Web View

The QWebEngineView is a powerful widget based on Chromium.

Key Steps:

  1. Import QWebEngineView.
  2. Import QUrl for handling URLs.
  3. Set the view as the Central Widget.
from PySide6.QtWebEngineWidgets import QWebEngineView
from PySide6.QtCore import QUrl

class BrowserWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("My PySide6 Browser")

        # Create the Web View
        self.browser = QWebEngineView()
        
        # Set Default URL
        self.browser.setUrl(QUrl("http://www.google.com"))
        
        # Set as Center of Window
        self.setCentralWidget(self.browser)

The Navigation Bar

We need a toolbar at the top. In Qt, we add QAction items to a QToolBar.

The Address Bar Logic

We need a text input (QLineEdit) to type URLs and sync it with the browser.

  • Input: When user hits enter, load the URL.
  • Feedback: When the website changes, update the text bar.
from PySide6.QtWidgets import QLineEdit

    # Inside __init__...

    self.url_bar.returnPressed.connect(self.navigate_to_url)
    nav_bar.addWidget(self.url_bar)

    # Sync URL changes

# Methods inside the Class
def navigate_to_url(self):
    url = self.url_bar.text()
    if not url.startswith("http"):
        url = "http://" + url
    self.browser.setUrl(QUrl(url))

def update_url(self, q):
    self.url_bar.setText(q.toString())
